# DocDiff

一个纯文档版本对比工具，支持 **TXT** 和 **DOCX** 格式，能够按段落粒度识别新增、删除、修改和未变更的内容，并以原文顺序混合展示修订情况。支持导出 **HTML** 和 **Markdown** 格式的详细报告。

本项目为100%  Vibe Coding 项目（含此README）。

## ✨ 功能特性

- **多格式支持**：支持 `.txt` 和 `.docx` 文件，自动识别。
- **段落级对比**：以自然段落为基本单位，保留原文结构。
- **相似度算法可选**：内置 TF‑IDF、BM25、Jaccard、Levenshtein 四种算法，可按需切换。
- **精确修订识别**：
  - 新增段落
  - 删除段落  
  - 修改段落（高亮显示具体增删内容）
  - 未变更段落
- **顺序混合展示**：所有修订按照段落在原文中的出现顺序排列，符合阅读习惯。
- **终端彩色输出**：使用 ANSI 颜色区分变更类型，视觉清晰。
- **可调阈值**：支持自定义相似度阈值，灵活控制匹配严格程度。
- **导出报告**：支持生成 **HTML**（简洁扁平风格）和 **Markdown** 格式的对比报告，便于存档和分享。
- **轻量级**：无需大型深度学习模型，纯 CPU 运行，响应迅速。

## 🚀 使用方法

### 基本用法

```bash
python main.py 旧文件路径 新文件路径
```

示例：
```bash
python main.py old_version.docx new_version.docx
```

### 指定相似度算法

```bash
python main.py old.txt new.txt --method tfidf
```

支持的算法：
- `tfidf`（默认）：TF‑IDF + 余弦相似度
- `bm25`：BM25 排序算法
- `jaccard`：Jaccard 集合相似度
- `levenshtein`：编辑距离相似度

### 调整相似度阈值

```bash
python main.py old.docx new.docx --threshold 0.4
```

阈值范围 `0.0 ~ 1.0`，默认 `0.3`。  
值越高，匹配要求越严格，更容易将实质相同但措辞有异的段落判定为“修改”而非“删除+新增”。

### 不显示终端详细差异内容

```bash
python main.py old.docx new.docx --no-detail
```

此时终端只显示每个段落的变更类型和编号，不展示具体内容/差异详情。

### 导出报告

```bash
# 只生成 HTML 报告
python main.py old.docx new.docx --report html

# 只生成 Markdown 报告
python main.py old.docx new.docx --report md

# 同时生成 HTML 和 Markdown 报告
python main.py old.docx new.docx --report both

# 指定输出目录
python main.py old.docx new.docx --report html --output-dir ./reports
```

报告默认命名格式为：`comparison_<旧文件名>_<新文件名>.html` 或 `.md`。

### 完整命令行参数

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `old_file` | - | 旧版本文件路径 | 必填 |
| `new_file` | - | 新版本文件路径 | 必填 |
| `--method` | `-m` | 相似度计算方法 | `tfidf` |
| `--threshold` | `-t` | 相似度阈值 (0-1) | `0.3` |
| `--no-detail` | - | 终端不显示详细差异内容 | `False` |
| `--report` | `-r` | 生成报告格式：`html`, `md`, `both` | 无 |
| `--output-dir` | `-o` | 报告输出目录 | 当前目录 |

## 🧠 相似度算法选择指南

| 方法 | 适用场景 | 特点 |
|------|----------|------|
| **TF‑IDF** | 大多数文本，特别是条款、法规等 | 平衡性好，对同义词和语序变化鲁棒，**默认推荐** |
| **BM25** | 段落长度差异大时 | 内建长度归一化，对长短不一的段落更公平 |
| **Jaccard** | 关键词重合度检测 | 简单快速，忽略词频和顺序，适合粗略去重 |
| **Levenshtein** | 短文本精确匹配 | 基于字符编辑距离，对微调敏感，不推荐作为主相似度 |

对于实施细则类文档，建议保留 **TF‑IDF** 或尝试 **BM25**。

## 📝 工作原理

1. **段落解析**  
   - TXT：按空行分割段落。  
   - DOCX：读取每个非空段落作为独立单元。

2. **预处理**  
   - 移除条款编号（如“第1条”、“1.”、“(1)”等）。  
   - 去除标点符号，仅保留中文字符和英文字母。

3. **相似度计算**  
   - 根据所选方法计算新旧段落之间的相似度矩阵。  
   - 使用贪心匹配算法为每个新段落找到最佳匹配旧段落。

4. **变更分类**  
   - 相似度 ≥ 0.85 且内容相同 → **未变更**  
   - 相似度 ≥ 0.5 或关键词重叠较多 → **修改**  
   - 相似度低于阈值且无关键词重叠 → **删除 + 新增**

5. **顺序重组**  
   - 加入段落原始顺序信息，将所有变更（删除、新增、修改、未变更）混合排序后输出。

6. **报告生成**  
   - HTML 报告：扁平化设计，支持差异高亮。  
   - Markdown 报告：纯文本，便于版本控制和纯文本环境阅读。

## ⚠️ 注意事项

- **文件编码**：仅支持 UTF-8 编码的 TXT 文件，DOCX 无编码问题。
- **段落定义**：TXT 中两个空行之间的内容视为一个段落；DOCX 中每个非空 Paragraph 对象视为一个段落。
- **中文支持**：算法已针对中文优化，但分词仅用于关键词提取（不是相似度计算的一部分）。
- **性能**：对于正常大小的文档（几百段），所有方法均可在数秒内完成。Levenshtein 在段落较长时可能稍慢。
